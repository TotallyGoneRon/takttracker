| Token | Value | Usage |

| xs | 4px | Icon gaps, badge internal padding |
| sm | 8px | Compact element spacing, thumbnail margin |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section padding, expanded detail area padding |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Major section breaks |

Exceptions:
- Touch targets: 48px minimum height on all tappable elements (existing pattern: `min-h-[48px]`)

## Interaction Contracts

### Entry Card (Collapsed State) — D-10

Layout: Single row, left-to-right:
1. Status color dot (8px circle, left edge)
2. Task name + zone name (stacked, flex-1)
3. Badge row (right-aligned): camera icon if photo exists, severity dot if set, percent number if set
4. Chevron icon (right edge, 20px, `text-gray-400`)

Touch target: entire card row is tappable to expand (min-h-[48px]).
Spacing: 16px horizontal padding, 12px vertical padding.

### Entry Card (Expanded State) — D-06

Expand animation: `max-height` transition, 200ms ease-out. No layout shift for sibling cards during animation.

Expanded panel content (vertical stack, 16px gap):
1. **Photo row**: thumbnail (64x64, rounded-lg, object-cover) on left + camera button on right. If no photo, show camera button only with label "Add Photo".
2. **Severity picker**: horizontal button group, 4 buttons equally spaced. One tap to set, tap same to unset. No selected state = no severity.
3. **Percent complete**: horizontal segmented buttons (5 segments: 0, 25, 50, 75, 100). Active segment uses `bg-blue-600 text-white`. Inactive uses `bg-gray-100 text-gray-700`.
4. **Notes field**: single-line text input (expandable to 3 lines via textarea). Placeholder: "Add a note..." Auto-save on blur with 500ms debounce.

Collapse: tap chevron again or tap a different entry card.

### Camera Capture — D-01, D-02, D-03

Trigger: tap camera icon button (`Camera` lucide icon, 20px, inside `Button` variant="outline" size="icon").
Implementation: hidden `<input type="file" accept="image/*" capture="environment">` triggered via ref click.
After capture: show `Skeleton` (64x64) immediately, upload async via `FormData` POST to `/api/site-walks/photos`.
On success: replace skeleton with `PhotoThumbnail` component.
On failure: show red border on skeleton area + "Upload failed. Tap to retry." text below.

### Photo Overlay — D-04, D-05

Trigger: tap the photo thumbnail on an entry card.
Layout: fixed fullscreen overlay (`fixed inset-0 z-50 bg-black/90`).
Content:
- Full-size image centered, `object-contain`, max dimensions `100vw x calc(100vh - 120px)`.
- Top bar: X close button (top-right, 44x44 hit area, white icon, `aria-label="Close photo"`).
- Bottom bar: two buttons centered, 16px gap.
  - "Retake" button: `Button` variant="outline" with `Camera` icon. White text/border on dark background.
  - "Delete" button: `Button` variant="destructive" with `Trash2` icon.

Close: tap X, tap outside the image area, or swipe down (optional enhancement).

### Delete Photo Confirmation — D-05

Trigger: tap "Delete" in photo overlay.
Pattern: inline confirmation within the overlay bottom bar. "Delete" button transforms to "Confirm Delete?" (red background) for 3 seconds, then reverts. Tap during confirmation window to delete. No modal dialog — too slow for field use.

## State Behaviors

### Photo Upload State Machine

```
idle -> capturing (user tapped camera)
capturing -> uploading (file selected from native camera)
uploading -> success (API returned thumbnail URL)
uploading -> error (API failed)
error -> uploading (user tapped retry)
success -> idle (thumbnail displayed)
```

### Severity Selection

- Unset: no button selected, all outlined
- Set: one button filled with its color, others outlined
- Toggle: tap active button to unset (returns to unset state)
- Persisted: saved immediately on change via PATCH to entry

### Percent Complete Selection

- Unset: no segment highlighted (distinct from "0%" which is an explicit selection)
- Set: one segment filled blue, others gray
- Toggle: tap active segment to unset
- Persisted: saved immediately on change via PATCH to entry

### Expanded/Collapsed Entry Cards

- Only one entry card expanded at a time (accordion behavior)
- Expanding card A auto-collapses any previously expanded card B
- Scroll into view: when expanding, scroll the expanded card to be fully visible if it would be clipped
